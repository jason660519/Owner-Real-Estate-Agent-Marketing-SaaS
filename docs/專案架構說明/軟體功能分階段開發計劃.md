# 房東物件管理語音 AI App - 軟體功能分階段開發計劃

本文件規劃 Web App 與 Mobile App 的整體功能演進路線圖，協助團隊聚焦優先級並逐步擴張能力。未列於各階段的功能請暫不開發。

---

## 1. 發展原則

*   以 AI 語音助理做為操作中樞，支援語音與文字雙模式互動。
*   所有資料與文件需在雲端可追蹤、可稽核，確保房東與租客資訊安全。
*   每一階段可獨立上線，並為下一階段鋪路；盡可能減少返工。

---

## 2. 發布里程碑概覽

| 階段 | 目標 | 核心成果 |
| :--- | :--- | :--- |
| **Phase 1** | 建立最小可行產品 (MVP) | 上架物件、生成行銷頁、維護租賃流程的最低限功能 |
| **Phase 2** | 擴充財務與維運能力 | 導入更完整的金流、財報、維修追蹤與多銀行帳戶管理 |
| **Phase 3** | 進階增值服務 | 圖像導覽、租金保障與教育支援，強化平台黏著度 |

## 4. 後續規劃建議

*   Phase 1 完成後需建立關鍵 KPI（如上架物件數、租金收款成功率），評估再進入 Phase 2。
*   金流與保險功能導入前，必須完成法遵評估與合作夥伴簽約流程。
*   進階導覽與租金保障屬高價值服務，建議與專業供應商合作試點後，再全面推廣。
