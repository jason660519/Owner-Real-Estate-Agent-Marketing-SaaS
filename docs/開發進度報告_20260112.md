## 2. 開發心得與感想 (Insights & Reflections)

- **跨平台樣式的細節**：React Native 雖然標榜 Write Once, Run Anywhere，但在 Web 端 (React Native Web) 的細節處理上仍需注意 CSS 屬性的對應，特別是陰影 (Shadows) 與佈局 (Layout) 在不同平台上的表現差異頗大。
- **開發伺服器的穩定性**：在使用 Expo 開發 Web 版時，若代碼出現嚴重語法錯誤 (Syntax Error)，原有的 Metro Bundler 進程容易終止或卡住。這時往往需要手動終止終端機並重新啟動 (`npx expo start --web`)，且需注意端口 (Port) 是否發生變化（如從 8081 變為 8082）。
- **AI 協作的精準度**：在進行大規模程式碼替換時，需特別小心括號與物件結構的完整性，避免因自動化工具的替換範圍誤差導致檔案損壞。
